# Security Policy

## Supported Versions

The latest `main` branch and the most recent tagged release receive security fixes.

## Reporting a Vulnerability

Do not open public GitHub issues for credential leaks, authentication bypasses, or command-execution vulnerabilities.

Instead, contact the maintainers privately and include:

- A clear description of the issue
- Reproduction steps
- Impact assessment
- Any required environment details

Until a dedicated security contact is published, use a private maintainer channel associated with the repository owner and avoid disclosing Telegram tokens, Claude links, or local filesystem details in plain text.
